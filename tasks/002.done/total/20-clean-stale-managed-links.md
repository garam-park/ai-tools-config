## 배경
원본에서 제거된 스킬의 심볼릭 링크가 도구별 대상 경로에 그대로 남는다. 사용자가 직접 만든 파일/디렉토리는 건드리지 않으면서 **스크립트가 만든 관리 링크만** 안전하게 정리하는 메커니즘이 필요.

## 작업

- 스크립트가 생성한 링크 목록을 manifest 파일(예: `~/.local/share/skills/.manifest.json`)로 기록
- 설치 시 이전 manifest와 현재 원본을 비교
- 이전에는 있지만 현재 원본에는 없는 항목 탐지
- 해당 항목이 여전히 스크립트가 만든 심링크일 때만 제거
- 사용자가 링크를 파일/디렉토리로 교체한 경우 제거하지 않고 경고
- 설치 성공 후 manifest를 현재 상태로 갱신

## 완료 조건
- 원본에서 스킬을 삭제한 뒤 설치 스크립트를 실행하면 관리 대상의 오래된 링크만 제거됨
- 사용자 소유 항목은 제거되지 않음
- manifest 자체가 손상/유실돼도 다음 실행에서 자가 복구
