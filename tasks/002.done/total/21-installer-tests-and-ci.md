## 배경
다른 두 모델은 단순 버그 수정에 집중했지만, codex는 **테스트 자동화 + CI**가 장기 유지보수의 핵심이라고 보았다. 현재는 P1(데이터 손실 위험) 변경들이 **자동 검증 없이 머지되는 상황**.

## 작업

### 테스트 범위
- 임시 디렉토리를 `HOME`으로 사용하는 테스트 환경 구성
- 다음 시나리오 커버:
  - 최초 설치 (대상 경로 비어있음)
  - 반복 설치 (멱등성)
  - 기존 파일/디렉토리 충돌 보호 (작업 01)
  - 잘못된 링크 교체
  - 원본 스킬 삭제 후 stale link 정리 (작업 20)
  - 기존 글로벌 지침 보호 + 백업 (작업 02)
  - 경로에 공백이 있는 경우
  - 마커 있는/없는 글로벌 지침 파일

### 정적 검사
- ShellCheck를 CI에서 실행
- `set -euo pipefail` + `shopt -s inherit_errexit` 일관성 검사

### CI
- GitHub Actions / 기타: PR마다 셸 검사 + 설치 테스트 실행
- README에 테스트 실행 방법 기록

## 완료 조건
- 실제 사용자 홈 디렉토리를 변경하지 않고 모든 설치 동작을 검증
- 데이터 손실 가능성이 있는 회귀를 테스트가 차단
- 새 커밋에서 셸 정적 검사와 설치 테스트가 자동 실행
