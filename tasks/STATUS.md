## 운영 규칙

- 작업 전에는 카드를 `tasks/001.todo/`에 두고 상태 체크박스로 진행 상황을 표시한다.
- 완료 조건과 검증을 모두 충족하면 체크박스를 채우고 `tasks/002.done/`의 같은 분류 경로로 이동한다.
- 별도의 `started`/`status` 헤더는 두지 않는다. 디렉토리 위치와 카드 체크박스를 단일 상태 원천으로 사용해 중복 상태가 어긋나는 일을 막는다.
- 작업 카드 관련 커밋에는 단일 작업이면 `(task NN)`, 묶음 작업이면 `(tasks NN,MM)` 토큰을 넣고 `git log --grep="task "`로 대응 관계를 확인한다.
