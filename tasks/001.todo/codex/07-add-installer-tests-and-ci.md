# 설치 스크립트 테스트와 정적 검사 추가

## 목적

설치 스크립트의 안전성과 멱등성을 자동으로 검증한다.

## 작업

- 임시 디렉터리를 `HOME`으로 사용하는 테스트 환경을 만든다.
- 최초 설치, 반복 설치, 기존 파일 충돌, 잘못된 링크 교체를 테스트한다.
- 원본 스킬 삭제 후 오래된 관리 링크가 정리되는지 테스트한다.
- 기존 전역 지침 보호와 백업 동작을 테스트한다.
- 경로에 공백이 있는 경우도 테스트한다.
- ShellCheck를 실행하는 CI를 추가한다.
- 테스트 실행 방법을 README에 기록한다.

## 완료 조건

- 실제 사용자 홈 디렉터리를 변경하지 않고 모든 설치 동작을 검증한다.
- 데이터 손실 가능성이 있는 회귀를 테스트가 차단한다.
- 새 커밋에서 셸 정적 검사와 설치 테스트가 자동 실행된다.
